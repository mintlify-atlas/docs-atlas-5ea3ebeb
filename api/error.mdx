`BroccoliError` is the unified error type returned by all Broccoli operations. It implements `std::error::Error` via the `thiserror` crate, providing ergonomic error handling and display formatting.

## Overview

All async operations in Broccoli return `Result<T, BroccoliError>`. The error variants are organized into logical groups:

- **Broker errors**: Connection and broker-level issues
- **Operation errors**: Publish, consume, acknowledge, reject, cancel failures
- **Data errors**: Serialization and position tracking failures
- **Backend-specific errors**: Redis, SurrealDB (feature-gated)
- **Other errors**: Job processing, queue status, timeouts, unimplemented features

## Error Variants

<Accordion title="Broker Errors">
  
  ### Broker(String)
  
  General broker operation error.
  
  ```rust
  #[error("Broker error: {0}")]
  Broker(String)
  ```
  
  **When it occurs:**
  - Broker connection failures
  - Connection pool initialization errors
  - Broker-specific protocol errors
  
  **How to handle:**
  ```rust
  match queue.consume("topic", None).await {
      Ok(msg) => { /* process */ },
      Err(BroccoliError::Broker(msg)) => {
          eprintln!("Broker error: {}", msg);
          // Possibly retry queue creation or check broker availability
      }
      _ => { /* other errors */ }
  }
  ```
  
  ---
  
  ### BrokerNonIdempotentOp(String)
  
  Concurrent operation that is not idempotent. **Should not be retried.**
  
  ```rust
  #[error("Broker error: non-idempotent operation {0}")]
  BrokerNonIdempotentOp(String)
  ```
  
  **When it occurs:**
  - Multiple concurrent reads on the same queue (SurrealDB)
  - Concurrent operations that conflict
  
  **How to handle:**
  ```rust
  match queue.try_consume("topic", None).await {
      Ok(Some(msg)) => { /* process */ },
      Err(BroccoliError::BrokerNonIdempotentOp(msg)) => {
          // Log and skip this attempt
          log::error!("Non-idempotent operation failed: {}", msg);
          // Try again on next iteration, but don't retry immediately
      }
      _ => { /* other errors */ }
  }
  ```
  
  <Warning>
    Do not retry `BrokerNonIdempotentOp` immediately. Wait for next cycle or use exponential backoff.
  </Warning>
  
  ---
  
  ### BrokerNonIdempotentRetriableOp(String)
  
  Concurrent operation that is not idempotent but **can be safely retried**.
  
  ```rust
  #[error("Broker error: non-idempotent retriable operation {0}")]
  BrokerNonIdempotentRetriableOp(String)
  ```
  
  **When it occurs:**
  - Multiple concurrent writes on the same table (SurrealDB)
  - Transient concurrent conflicts that resolve with retry
  
  **How to handle:**
  ```rust
  match queue.publish("topic", None, &msg, None).await {
      Ok(published) => { /* use published message */ },
      Err(BroccoliError::BrokerNonIdempotentRetriableOp(msg)) => {
          // Safe to retry
          log::warn!("Retryable non-idempotent operation: {}", msg);
          tokio::time::sleep(Duration::from_millis(100)).await;
          // Try again
          let _ = queue.publish("topic", None, &msg, None).await;
      }
      _ => { /* other errors */ }
  }
  ```
  
  <Tip>
    Use exponential backoff when retrying these operations to reduce contention.
  </Tip>
  
  ---
  
  ### ConnectionTimeout(u32)
  
  Failed to establish connection within timeout period.
  
  ```rust
  #[error("Connection timeout after {0} retries")]
  ConnectionTimeout(u32)
  ```
  
  **When it occurs:**
  - Broker unreachable
  - Network connectivity issues
  - Broker startup delays
  
  **Parameters:**
  - `u32` - Number of retry attempts made before timing out
  
  **How to handle:**
  ```rust
  match BroccoliQueue::builder("redis://localhost").build().await {
      Ok(queue) => { /* use queue */ },
      Err(BroccoliError::ConnectionTimeout(retries)) => {
          eprintln!("Connection failed after {} retries", retries);
          eprintln!("Check broker availability and network connectivity");
          std::process::exit(1);
      }
      _ => { /* other errors */ }
  }
  ```

</Accordion>

<Accordion title="Operation Errors">
  
  ### Publish(String)
  
  Failed to publish message to broker.
  
  ```rust
  #[error("Failed to publish message: {0}")]
  Publish(String)
  ```
  
  **When it occurs:**
  - Broker temporarily unavailable
  - Message encoding fails
  - Broker-side write errors
  
  **How to handle:**
  ```rust
  match queue.publish("jobs", None, &payload, None).await {
      Ok(msg) => println!("Published {}", msg.task_id),
      Err(BroccoliError::Publish(reason)) => {
          eprintln!("Publish failed: {}", reason);
          // Decide: retry, queue locally, or fail fast
      }
      _ => { /* other errors */ }
  }
  ```
  
  ---
  
  ### Consume(String)
  
  Failed to consume message from broker.
  
  ```rust
  #[error("Failed to consume message: {0}")]
  Consume(String)
  ```
  
  **When it occurs:**
  - Broker connection lost mid-operation
  - Queue doesn't exist
  - Message deserialization fails
  
  **How to handle:**
  ```rust
  match queue.consume("jobs", None).await {
      Ok(msg) => { /* process */ },
      Err(BroccoliError::Consume(reason)) => {
          log::error!("Consume failed: {}", reason);
          // For process_messages, these are handled internally
          // For manual consume, reconnect and retry
      }
      _ => { /* other errors */ }
  }
  ```
  
  ---
  
  ### Acknowledge(String)
  
  Failed to acknowledge (remove) processed message.
  
  ```rust
  #[error("Failed to acknowledge message: {0}")]
  Acknowledge(String)
  ```
  
  **When it occurs:**
  - Broker disconnected after consuming
  - Message already acknowledged
  - Broker state corruption
  
  **How to handle:**
  ```rust
  // Manually acknowledging (if handler_ack disabled)
  match queue.acknowledge("jobs", message).await {
      Ok(_) => println!("Message acknowledged"),
      Err(BroccoliError::Acknowledge(reason)) => {
          log::error!("Failed to acknowledge: {}", reason);
          // Message will be retried by broker
          // Consider alerting ops
      }
      _ => { /* other errors */ }
  }
  ```
  
  <Tip>
    If using `handler_ack: true` in `ConsumeOptions`, acknowledgment is automatic and most failures are logged but not fatal.
  </Tip>
  
  ---
  
  ### Reject(String)
  
  Failed to reject (move to retry/failed queue) message.
  
  ```rust
  #[error("Failed to reject message: {0}")]
  Reject(String)
  ```
  
  **When it occurs:**
  - Broker disconnected
  - Failed queue doesn't exist
  - Attempt count corruption
  
  **How to handle:**
  ```rust
  match queue.reject("jobs", message).await {
      Ok(_) => println!("Message rejected and requeued"),
      Err(BroccoliError::Reject(reason)) => {
          log::error!("Failed to reject message: {}", reason);
          // Message state is uncertain
          // May need manual intervention
      }
      _ => { /* other errors */ }
  }
  ```
  
  ---
  
  ### Cancel(String)
  
  Failed to cancel (permanently remove) message.
  
  ```rust
  #[error("Failed to cancel message: {0}")]
  Cancel(String)
  ```
  
  **When it occurs:**
  - Broker disconnected
  - Message ID not found
  - Insufficient permissions
  
  **How to handle:**
  ```rust
  match queue.cancel("jobs", message_id).await {
      Ok(_) => println!("Message cancelled"),
      Err(BroccoliError::Cancel(reason)) => {
          log::warn!("Failed to cancel message: {}", reason);
          // Message may still process
      }
      _ => { /* other errors */ }
  }
  ```

</Accordion>

<Accordion title="Data Errors">
  
  ### Serialization(serde_json::Error)
  
  Failed to serialize or deserialize message payload.
  
  ```rust
  #[error("Serialization error: {0}")]
  Serialization(#[from] serde_json::Error)
  ```
  
  **When it occurs:**
  - Payload type doesn't implement `Serialize`/`Deserialize`
  - Payload contains non-serializable types
  - JSON data corruption in broker
  - Type mismatch between publish and consume
  
  **How to handle:**
  ```rust
  #[derive(Serialize, Deserialize, Clone)]
  struct MyPayload { /* ... */ }
  
  match queue.consume::<MyPayload>("jobs", None).await {
      Ok(msg) => { /* process */ },
      Err(BroccoliError::Serialization(err)) => {
          eprintln!("Type mismatch or corruption: {}", err);
          // Check your type definitions
          // Possible fix: ensure all payload fields are serializable
      }
      _ => { /* other errors */ }
  }
  ```
  
  <Warning>
    Serialization errors indicate either a programming error (missing Serialize derive) or data corruption. Check your types and data integrity.
  </Warning>
  
  ---
  
  ### GetMessagePosition(String)
  
  Failed to retrieve message position in queue.
  
  ```rust
  #[error("Failed to get message position: {0}")]
  GetMessagePosition(String)
  ```
  
  **When it occurs:**
  - Feature not supported by broker (e.g., RabbitMQ)
  - Message not found in queue
  - Broker state inconsistency
  
  **How to handle:**
  ```rust
  match queue.get_message_position(message_id).await {
      Ok(pos) => println!("Message is at position {}", pos),
      Err(BroccoliError::GetMessagePosition(reason)) => {
          if reason.contains("not implemented") {
              // Feature not available on this broker
              log::info!("Position tracking not supported");
          } else {
              log::error!("Failed to get position: {}", reason);
          }
      }
      _ => { /* other errors */ }
  }
  ```

</Accordion>

  ### Job(String)
  
  Error during job/message processing.
  
  ```rust
  #[error("Job error: {0}")]
  Job(String)
  ```
  
  **When it occurs:**
  - Handler function returns error
  - Business logic failure

  **How to handle:**
  ```rust
  queue.process_messages("jobs", None, None, |msg| async {
      match my_handler(&msg).await {
          Ok(_) => Ok(()),
          Err(e) => {
              // Wrap business errors
              Err(BroccoliError::Job(e.to_string()))
          }
      }
  }).await?;
  ```

## Error Handling Patterns

### Pattern 1: Match and Log

```rust
match queue.consume("topic", None).await {
    Ok(msg) => handle_message(msg).await,
    Err(BroccoliError::Consume(msg)) => {
        log::error!("Consume failed: {}", msg);
    }
    Err(BroccoliError::Serialization(err)) => {
        log::error!("Type mismatch: {}", err);
    }
    Err(e) => {
        log::error!("Unexpected error: {}", e);
    }
}
```

### Pattern 2: Blanket Error Propagation

```rust
async fn process_workflow() -> Result<(), BroccoliError> {
    let queue = BroccoliQueue::builder("redis://localhost").build().await?;
    
    let msg = queue.consume("jobs", None).await?; // Propagate any error
    handle_message(&msg).await?;
    queue.acknowledge("jobs", msg).await?;
    
    Ok(())
}
```

### Pattern 3: Structured Error Handling with Context

```rust
match operation().await {
    Ok(result) => Ok(result),
    Err(e) => {
        let context = format!("Operation failed for queue 'jobs': {}", e);
        log::error!("{}", context);
        Err(BroccoliError::Broker(context))
    }
}
```

### Pattern 4: Handler with Error Recovery

```rust
queue.process_messages("jobs", Some(3), None, |msg: BrokerMessage<JobPayload>| async move {
    match process_job(&msg.payload).await {
        Ok(_) => Ok(()),
        Err(e) => {
            if msg.attempts < 2 {
                // Retry for transient errors
                log::warn!("Job failed (attempt {}), will retry: {}", msg.attempts + 1, e);
                Err(BroccoliError::Job(e.to_string()))
            } else {
                // Give up on permanent errors
                log::error!("Job failed on final attempt: {}", e);
                Err(BroccoliError::Job(e.to_string()))
            }
        }
    }
}).await?;
```

## Error Display and Debugging

`BroccoliError` implements `std::fmt::Display` for human-readable output:

```rust
match operation().await {
    Ok(result) => println!("Success: {}", result),
    Err(e) => {
        println!("{}", e);           // Display trait
        println!("{:?}", e);         // Debug trait
        println!("Source: {:?}", e.source()); // Error trait
    }
}
```

## Integration with Error Handling Libraries

### With `anyhow`:

```rust
use anyhow::Context;

let queue = BroccoliQueue::builder("redis://localhost")
    .build()
    .await
    .context("Failed to connect to broker")?;

let msg = queue
    .consume("jobs", None)
    .await
    .context("Failed to consume from jobs queue")?;
```

### With `thiserror` custom error:

```rust
use thiserror::Error;

#[derive(Error, Debug)]
#[error("Application error: {0}")]
pub enum AppError {
    Queue(#[from] BroccoliError),
    Custom(String),
}

async fn my_handler() -> Result<(), AppError> {
    let queue = BroccoliQueue::builder("redis://localhost").build().await?;
    Ok(())
}
```

## Summary Table

| Error | Retryable | When | Action |
|-------|-----------|------|--------|
| `Broker` | Maybe | Connection issues | Check broker, retry connection |
| `BrokerNonIdempotentOp` | No | Concurrent reads | Log and skip, wait for next cycle |
| `BrokerNonIdempotentRetriableOp` | Yes | Concurrent writes | Retry with backoff |
| `Publish` | Maybe | Write failed | Check broker, retry |
| `Consume` | Maybe | Read failed | Check broker, retry |
| `Acknowledge` | No | Already ack'd | Log, message retries |
| `Reject` | No | Reject failed | Alert ops, uncertain state |
| `Cancel` | Maybe | Msg not found | Log, may still process |
| `Serialization` | No | Type mismatch | Fix types, fix data |
| `GetMessagePosition` | No | Not supported | Check broker features |
| `Redis` | Maybe | Redis error | Check Redis server |
| `SurrealDB` | Maybe | DB error | Check database |
| `Job` | Up to retry limit | Handler failed | RetryStrategy decides |
| `QueueStatus` | Maybe | Status read failed | Check management feature |
| `ConnectionTimeout` | No | Timeout | Check broker availability |
| `NotImplemented` | No | Feature unavailable | Use alternative approach |

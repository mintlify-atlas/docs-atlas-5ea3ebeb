**Install Broccoli and configure your message broker backend.** The library supports Redis (default), RabbitMQ, and SurrealDB, with flexible feature flags for minimal dependencies.

## Basic Installation

Add to your `Cargo.toml`:

```toml
[dependencies]
broccoli_queue = "0.4.6"
```

This installs Broccoli with the **Redis broker enabled by default**. You'll also need a runtime:

```toml
[dependencies]
broccoli_queue = "0.4.6"
tokio = { version = "1.42", features = ["full"] }
serde = { version = "1.0", features = ["derive"] }
serde_json = "1.0"
```

## Choosing Your Broker

Broccoli requires **at least one broker feature** to be enabled. You cannot use Broccoli with zero broker features—attempting to compile will produce a `compile_error!`.

### Redis (Default)

Perfect for: Development, small-to-medium workloads, familiar tooling.

```toml
[dependencies]
broccoli_queue = "0.4.6"  # redis feature enabled by default
```

**Connection URL:** `redis://host:port`

```rust
let queue = BroccoliQueue::builder("redis://localhost:6379").build().await?;
```

### RabbitMQ

Perfect for: Enterprise deployments, complex routing, high reliability requirements.

```toml
[dependencies]
broccoli_queue = { version = "0.4.6", features = ["rabbitmq"], default-features = false }
tokio = { version = "1.42", features = ["full"] }
```

**Connection URL:** `amqp://user:password@host:port`

```rust
let queue = BroccoliQueue::builder("amqp://guest:guest@localhost:5672").build().await?;
```

### SurrealDB

Perfect for: Embedded scenarios, in-memory testing, multi-model queries.

```toml
[dependencies]
broccoli_queue = { version = "0.4.6", features = ["surrealdb"], default-features = false }
```

**Connection URLs:**
- WebSocket (remote): `ws://host:port`
- In-memory (local): `mem://`

```rust
// Remote SurrealDB
let queue = BroccoliQueue::builder("ws://localhost:8000").build().await?;

// In-memory SurrealDB (for testing)
let queue = BroccoliQueue::builder("mem://").build().await?;
```

### Multiple Brokers

Enable multiple brokers in a single binary:

```toml
[dependencies]
broccoli_queue = { version = "0.4.6", features = ["redis", "rabbitmq"] }
```

Then select at runtime based on environment or configuration:

```rust
let url = if use_rabbitmq {
    "amqp://localhost:5672"
} else {
    "redis://localhost:6379"
};

let queue = BroccoliQueue::builder(url).build().await?;
```

## Optional Features

### Management API

Enable inspection and management of queue state:

```toml
[dependencies]
broccoli_queue = { version = "0.4.6", features = ["redis", "management"] }
```

Provides APIs to:
- Query queue length
- Inspect job status
- Cancel pending jobs
- Monitor worker health

### Test Fairness

Enable fairness queue testing for benchmarking:

```toml
[dependencies]
broccoli_queue = { version = "0.4.6", features = ["redis", "test-fairness"] }
```

Used for validating equitable task distribution across workers.

## Runtime Requirements

### Tokio

Broccoli requires the **Tokio async runtime** with full features:

```toml
[dependencies]
tokio = { version = "1.42", features = ["full"] }
```

Minimal Tokio feature set (recommended for production):

```toml
tokio = { version = "1.42", features = ["rt-multi-thread", "macros", "io-util", "time", "net"] }
```

### Serialization

Broccoli uses **serde** for message serialization. You must enable the derive feature:

```toml
[dependencies]
serde = { version = "1.0", features = ["derive"] }
serde_json = "1.0"
```

All message types must derive `Serialize` and `Deserialize`:

```rust
use serde::{Deserialize, Serialize};

#[derive(Serialize, Deserialize)]
struct MyTask {
    id: String,
    data: String,
}
```

## Initializing the Queue

Once installed and configured, initialize `BroccoliQueue` with your broker URL:

```rust
use broccoli_queue::queue::BroccoliQueue;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let queue = BroccoliQueue::builder("redis://localhost:6379")
        .pool_connections(10)
        .failed_message_retry_strategy(Default::default())
        .build()
        .await?;

    // Queue is ready to use
    Ok(())
}
```

## Verification

Test your installation:

```bash
cargo build
```

If you get a `compile_error` about missing broker features, ensure at least one broker is enabled:

```toml
# ❌ This will fail to compile
broccoli_queue = { version = "0.4.6", default-features = false }

# ✅ This works
broccoli_queue = { version = "0.4.6", features = ["redis"] }
```

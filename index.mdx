Broccoli is a Rust-native message queue library inspired by Celery, designed for async job processing with Tokio. It provides a unified API across Redis, RabbitMQ, and SurrealDB backends — publish typed messages, consume them concurrently, and handle failures with configurable retry strategies.

## Why Broccoli?

Broccoli brings Celery-style task queue ergonomics to the Rust ecosystem — with compile-time type safety, async-first design, and zero boilerplate message serialization.

## Get Started

<Steps>
  <Step title="Add the dependency">
    Add `broccoli_queue` to your `Cargo.toml` with your chosen broker feature flag.
    ```toml Cargo.toml
    [dependencies]
    broccoli_queue = { version = "0.4", features = ["redis"] }
    ```
  </Step>
  <Step title="Build the queue">
    Connect to your broker using the builder pattern.
    ```rust
    use broccoli_queue::queue::BroccoliQueue;

    let queue = BroccoliQueue::builder("redis://localhost:6379")
        .pool_connections(5)
        .failed_message_retry_strategy(Default::default())
        .build()
        .await?;
    ```
  </Step>
  <Step title="Publish a message">
    Publish any serializable struct to a named topic.
    ```rust
    queue.publish("jobs", None, &my_payload, None).await?;
    ```
  </Step>
  <Step title="Process messages">
    Start a concurrent worker loop with a typed async handler.
    ```rust
    queue.process_messages("jobs", Some(4), None, |msg| async move {
        println!("Got job: {:?}", msg.payload);
        Ok(())
    }).await?;
    ```
  </Step>
</Steps>

<Note>
  Broccoli requires at least one broker feature to be enabled (`redis`, `rabbitmq`, or `surrealdb`). The `redis` feature is enabled by default.
</Note>
